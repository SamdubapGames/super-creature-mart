🗺️ path.js 테스트

path.js 파일을 열어서 함수를 구현하고, 이 html 파일을 브라우저에서 열면 결과가 나옵니다.

개발자 도구 (F12 → Console)에서도 확인할 수 있어요.

💡 findPartIndex(parts, targetPart) — 부위 배열에서 특정 부위의 인덱스 반환
💡 generateRoute(parts, n) — 랜덤 경로 배열 생성
💡 checkMatch(part, item, parts, items) — 부위+아이템 짝이 맞는지 판정
💡 useItem(inventory, itemName) — 인벤토리에서 아이템 제거
💡 playTurn(part, chosenItem, inventory, parts, items) — 한 턴 전체 처리 → { success, message }

🔍 Function 1: findPartIndex

테스트 대기중...
테스트가 ❌이면 무슨 뜻인지 보기
인덱스가 1씩 크게 나오면 — for문이 0이 아닌 1부터 시작하고 있습니다.
→ for (let i = 0; ...) 0부터 시작해야 배열 인덱스와 일치합니다.

없는 부위인데 -1이 안 나오면 — for문이 끝난 뒤 return이 없습니다.
못 찾으면 for문 밖에서 -1을 return해야 합니다.
→ return -1; 이 for문 바깥(함수 맨 끝)에 있는지 확인하세요.

숫자가 아닌 다른 값이 나오면 — 인덱스 대신 부위 이름 자체를 return하고 있습니다.
→ return i; (parts[i]가 아닙니다!)

원본 parts 배열 부작용 검사가 실패하면 — parts 배열을 수정하고 있습니다.
→ parts는 읽기만 해야 합니다. splice나 push를 쓰지 마세요.

🎲 Function 2: generateRoute

테스트 대기중...
테스트가 ❌이면 무슨 뜻인지 보기
길이가 틀리면 — for문이 n번 돌지 않고 있습니다.
→ for (let i = 0; i < n; i++) 에서 n이 맞는지 확인하세요.

유효하지 않은 값이 나오면 — 인덱스 범위가 parts 배열을 벗어나고 있습니다.
Math.random() * 10 같은 고정 숫자를 쓰면 범위를 벗어납니다.
→ Math.floor(Math.random() * parts.length) 로 parts 길이에 맞춰야 합니다.

랜덤 확인에서 1종류만 나오면 — Math.random()을 for문 밖에서 한 번만 계산하고 재사용하고 있습니다.
→ let idx = Math.floor(Math.random() * parts.length) 이 줄이 for문 안에 있어야 합니다.

원본 parts 배열 부작용 검사가 실패하면 — parts 배열에서 항목을 제거하고 있습니다.
경로는 같은 부위가 반복될 수 있습니다. parts는 건드리지 마세요.
→ route.push(parts[idx]) 만 하고 parts.splice는 쓰지 않습니다.

🎯 Function 3: checkMatch

테스트 대기중...
테스트가 ❌이면 무슨 뜻인지 보기
올바른 짝인데 false가 나오면 — findPartIndex를 호출하지 않거나 인덱스를 잘못 쓰고 있습니다.
findPartIndex(parts, part) 로 인덱스를 찾은 뒤 items[인덱스] === item 을 확인해야 합니다.
→ let idx = findPartIndex(parts, part); if (idx !== -1 && items[idx] === item) return true;

틀린 짝인데 true가 나오면 — 부위만 확인하고 아이템은 확인하지 않고 있습니다.
→ 인덱스를 찾은 뒤 items[인덱스] === item 도 확인해야 합니다.

없는 부위인데 오류가 나거나 이상한 값이 나오면 — findPartIndex가 -1을 반환할 때 처리가 없습니다.
→ 인덱스가 -1 이면 바로 return false; 해야 합니다.

원본 배열 부작용 검사가 실패하면 — parts나 items 배열을 수정하고 있습니다.
→ 배열은 읽기만 해야 합니다.

🎒 Function 4: useItem

테스트 대기중...
테스트가 ❌이면 무슨 뜻인지 보기
꺼낸 후 배열이 안 줄어들면 — splice를 쓰지 않았거나 위치가 틀렸습니다.
→ inventory.splice(i, 1) 이 if문 안, return 바로 앞에 있어야 합니다.

맨 앞 항목 제거 후 나머지가 안 밀리면 — splice 대신 inventory[i] = undefined 같은 걸 쓰고 있습니다.
splice는 그 자리를 완전히 없애고 뒤 항목을 앞으로 당겨줍니다.
→ inventory.splice(i, 1) 만 쓰면 됩니다.

없는 아이템인데 배열이 바뀌면 — splice를 if문 밖에서 무조건 호출하고 있습니다.
→ splice는 아이템을 찾았을 때(if문 안에서)만 호출해야 합니다.

같은 아이템이 2개인데 2개 다 사라지면 — splice 후에 바로 return하지 않아서 for문이 계속 돌고 있습니다.
→ splice와 return이 같은 if 블록 안에 있어야 합니다. splice 직후 함수를 끝내세요.

⚔️ Function 5: playTurn

테스트 대기중...
테스트가 ❌이면 무슨 뜻인지 보기
반환값이 객체가 아니면 — 문자열이나 true/false만 return하고 있습니다.
→ return { success: true, message: "..." } 처럼 객체를 반환해야 합니다.

success 또는 message 필드가 없으면 — 반환 객체의 키 이름이 다릅니다.
→ 키 이름이 정확히 success, message 여야 합니다. 오타를 확인하세요.

성공인데 success가 false로 나오면 — checkMatch를 호출하지 않거나 결과를 반대로 쓰고 있습니다.
→ if (checkMatch(part, chosenItem, parts, items)) 가 true일 때 success: true를 반환하세요.

성공 후 아이템이 인벤토리에서 안 사라지면 — useItem을 호출하지 않았습니다.
→ 성공했을 때 useItem(inventory, chosenItem)을 반드시 호출하세요.

실패 시 인벤토리가 바뀌면 — useItem을 성공/실패 구분 없이 항상 호출하고 있습니다.
→ useItem은 checkMatch가 true일 때(성공 분기)에서만 호출해야 합니다.
