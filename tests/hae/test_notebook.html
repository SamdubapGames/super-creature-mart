📓 notebook.js 테스트

notebook.js 파일을 열어서 함수를 구현하고, 이 html 파일을 브라우저에서 열면 결과가 나옵니다.

개발자 도구 (F12 → Console)에서도 확인할 수 있어요.

💡 formatTipLine(part, item) — "입 → 떡" 한 줄짜리 문자열 만들기
💡 showNotebook(parts, items) — formatTipLine 조합, 팁 목록 전체를 한 문자열로 반환

✍️ Function 1: formatTipLine

테스트 대기중...
테스트가 ❌이면 무슨 뜻인지 보기
"입 → 떡"이 안 나오면 — 문자열 조합 방식을 확인하세요.
part + " → " + item 처럼 화살표(→) 양쪽에 공백이 있어야 합니다.
→ return part + " → " + item; 이 한 줄로 완성됩니다.

"→" 기호가 없다고 나오면 — 일반 하이픈(-)이나 =>를 쓰고 있습니다.
→ 화살표는 특수문자 → 입니다. 복사해서 쓰세요.

part가 뒤, item이 앞에 오면 — 매개변수 순서가 바뀌었습니다.
함수 선언 function formatTipLine(part, item)에서
첫 번째 인자가 part, 두 번째가 item입니다.
→ part + " → " + item 순서를 확인하세요.

return이 없다고 나오면 — console.log만 쓰고 return을 빠뜨렸습니다.
→ 함수 안에 return 키워드가 반드시 있어야 합니다.

📋 Function 2: showNotebook

테스트 대기중...
테스트가 ❌이면 무슨 뜻인지 보기
팁이 하나만 나오거나 빠지면 — for문이 배열 전체를 돌지 않고 있습니다.
→ for (let i = 0; i < parts.length; i++) 로 끝까지 돌아야 합니다.

줄바꿈(\n)이 없다고 나오면 — 각 줄을 그냥 +로 붙이거나 공백으로 이어붙이고 있습니다.
→ 각 항목 사이에 "\n"을 넣어야 합니다. 배열에 push 하고 마지막에 join("\n") 써도 됩니다.

순서가 뒤집혀 나오면 — for문을 뒤에서 앞으로 돌고 있습니다.
→ i = 0부터 parts.length 미만까지 순서대로 돌면 됩니다.

formatTipLine 결과가 반영 안 되면 — showNotebook 안에서 formatTipLine을 호출하지 않고 직접 문자열을 만들고 있습니다.
→ 반드시 formatTipLine(parts[i], items[i])을 호출해서 결과를 이어붙이세요.

원본 배열 부작용 검사가 실패하면 — parts나 items 배열에 뭔가 추가하거나 제거하고 있습니다.
→ 배열은 읽기만 하고, 새 문자열 result에만 이어붙여야 합니다.
