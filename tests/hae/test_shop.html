🛒 shop.js 테스트

shop.js 파일을 열어서 함수를 구현하고, 이 html 파일을 브라우저에서 열면 결과가 나옵니다.

개발자 도구 (F12 → Console)에서도 확인할 수 있어요.

💡 findItemIndex(names, targetName) — 이름 배열에서 인덱스 찾기
💡 getStockLabel(stock) — "(품절)" 또는 "(재고: N개)"
💡 formatShopLine(number, name, price, stockLabel) — "1. 떡 — 100원 (재고: 3개)"
💡 showShopList(names, prices, stock) — 전체 목록 문자열
💡 isAffordable(price, budget) — 예산 확인 → true/false
💡 buyItem(name, budget, names, prices, stock) — 구매 처리 → 결과 메시지
💡 calcTotal(cart, names, prices) — 장바구니 합산 → 총액
💡 showReceipt(cart, names, prices) — 영수증 문자열 생성
💡 shopSession(shoppingList, budget, names, prices, stock) — 전체 흐름 (완성본)

🔍 Function 0: findItemIndex

테스트 대기중...
테스트가 ❌이면 무슨 뜻인지 보기
인덱스가 1씩 크게 나오면 — for문이 0이 아닌 1부터 시작하고 있습니다.
→ for (let i = 0; ...) 0부터 시작해야 배열 인덱스와 일치합니다.

없는 이름인데 -1이 안 나오면 — for문이 끝난 뒤 return이 없습니다.
→ return -1; 이 for문 바깥(함수 맨 끝)에 있는지 확인하세요.

숫자가 아닌 다른 값이 나오면 — 인덱스 대신 이름 자체를 return하고 있습니다.
→ return i; (names[i]가 아닙니다!)

원본 names 배열 부작용 검사가 실패하면 — names 배열을 수정하고 있습니다.
→ names는 읽기만 해야 합니다. splice나 push를 쓰지 마세요.

🏷️ Function 1: getStockLabel

테스트 대기중...
테스트가 ❌이면 무슨 뜻인지 보기
"(품절)"이 안 나오면 — stock이 0일 때의 분기가 없습니다.
→ if (stock === 0) return "(품절)"; 을 먼저 처리하세요.

"(재고: 3개)"에서 숫자가 다르게 나오면 — stock 값을 문자열에 끼워넣지 않고 고정 숫자를 쓰고 있습니다.
→ "(재고: " + stock + "개)" 처럼 변수를 직접 넣어야 합니다.

괄호나 콜론, 공백이 틀리면 — 형식이 정확히 맞아야 합니다.
→ (재고: N개) — 여는 괄호, "재고", 콜론, 공백, 숫자, "개", 닫는 괄호 순서입니다.

📝 Function 2: formatShopLine

테스트 대기중...
테스트가 ❌이면 무슨 뜻인지 보기
"1. 떡 — 100원 (재고: 3개)"가 안 나오면 — 구분자나 공백이 다릅니다.
점(.), 줄표(—), "원" 뒤 공백을 정확히 확인하세요.
→ number + ". " + name + " — " + price + "원 " + stockLabel

stockLabel이 이상하게 나오면 — stockLabel 파라미터는 이미 getStockLabel의 결과 문자열입니다.
→ stockLabel을 그대로 이어붙이면 됩니다. showShopList에서 getStockLabel을 호출하고 그 결과를 formatShopLine에 넘깁니다.

가격 뒤에 "원"이 없으면 — 숫자만 붙이고 "원"을 빠뜨렸습니다.
→ price + "원" 으로 붙여야 합니다.

📜 Function 3: showShopList

테스트 대기중...
테스트가 ❌이면 무슨 뜻인지 보기
상품이 하나만 나오거나 빠지면 — for문이 배열 전체를 돌지 않고 있습니다.
→ for (let i = 0; i < names.length; i++) 로 끝까지 돌아야 합니다.

번호가 0부터 시작하면 — i를 그대로 formatShopLine에 넘기고 있습니다.
→ formatShopLine(i + 1, ...) 으로 넘기세요.

줄바꿈(\n)이 없다고 나오면 — 항목들을 그냥 이어 붙이고 있습니다.
→ 첫 번째 줄은 \n 없이, 나머지는 앞에 "\n"을 붙여서 이어붙이세요.

품절 표시가 없으면 — getStockLabel을 호출하지 않고 있습니다.
→ getStockLabel(stock[i]) 를 호출해서 재고 라벨을 만들고 formatShopLine(i+1, names[i], prices[i], stockLabel) 에 넘기세요.

원본 배열 부작용 검사가 실패하면 — 배열을 수정하고 있습니다.
→ 배열은 읽기만 해야 합니다. push나 splice를 쓰지 마세요.

💰 Function 4: isAffordable

테스트 대기중...
테스트가 ❌이면 무슨 뜻인지 보기
딱 맞는 예산(9000, 9000)에서 false가 나오면 — <를 쓰고 있습니다. 등호가 빠졌습니다.
→ price <= budget 로 등호를 포함해야 합니다.

예산이 충분한데 false가 나오면 — 인자 순서가 바뀌었습니다. price와 budget을 거꾸로 비교하고 있습니다.
→ 함수 선언 isAffordable(price, budget) — price가 첫 번째, budget이 두 번째입니다.

반환값이 boolean이 아니면 — 숫자나 문자열을 return하고 있습니다.
→ return price <= budget; 비교 연산 자체가 true/false를 반환합니다.

🛍️ Function 5: buyItem

테스트 대기중...
테스트가 ❌이면 무슨 뜻인지 보기
없는 아이템인데 메시지가 안 나오면 — findItemIndex 결과를 체크하지 않고 있습니다.
→ findItemIndex(names, name) 가 -1이면 "없는 아이템입니다" 를 return 하세요.

품절인데 "품절" 메시지가 안 나오면 — 재고 확인을 하지 않고 있습니다.
→ if (stock[인덱스] === 0) 으로 품절을 체크하고 메시지를 반환하세요.

예산 부족인데 구매가 되면 — isAffordable을 호출하지 않거나 결과를 반대로 쓰고 있습니다.
→ if (!isAffordable(prices[인덱스], budget)) 로 예산 부족일 때 early return하세요.

구매 후 재고가 안 줄어들면 — stock[인덱스]를 감소시키지 않았습니다.
→ 구매 성공 시 stock[인덱스]-- 를 해야 합니다.

🧮 Function 6: calcTotal

테스트 대기중...
테스트가 ❌이면 무슨 뜻인지 보기
합계가 0으로 나오면 — total 누적을 하지 않고 있습니다.
→ let total = 0; 후 findItemIndex(names, cart[i]) 로 인덱스를 찾고 total += prices[인덱스]; 를 for문 안에서 반복하세요.

빈 장바구니에서 0이 안 나오면 — 초기값이 0이 아닌 다른 값으로 설정되어 있습니다.
→ let total = 0; 으로 시작해야 합니다.

합계가 더 크거나 작게 나오면 — findItemIndex로 찾은 인덱스가 잘못되었거나 prices를 쓰지 않고 있습니다.
→ prices[인덱스] 를 더해야 합니다.

원본 cart 배열 부작용 검사가 실패하면 — cart 배열을 수정하고 있습니다.
→ cart는 읽기만 해야 합니다. splice나 pop을 쓰지 마세요.

🧾 Function 7: showReceipt

테스트 대기중...
테스트가 ❌이면 무슨 뜻인지 보기
문자열이 안 나오면 — return이 없거나 객체를 return하고 있습니다.
→ 영수증은 문자열 하나여야 합니다. return result; 형태로 반환하세요.

상품명이 포함되지 않으면 — cart를 순회하지 않고 있습니다.
→ for문으로 cart를 돌면서 cart[i]와 prices[findItemIndex(names, cart[i])]를 영수증 문자열에 이어붙이세요.

총액이 포함되지 않으면 — calcTotal을 호출하지 않았습니다.
→ calcTotal(cart, names, prices) 로 합계를 구하고 영수증 문자열에 포함시키세요.

RECEIPT_HEADER 나 RECEIPT_FOOTER 가 없으면 — 영수증 머리말/꼬리말을 빠뜨렸습니다.
→ result = RECEIPT_HEADER 로 시작하고 result += "\n\n" + RECEIPT_FOOTER 로 끝내세요.

빈 장바구니 영수증에서 오류가 나면 — cart가 빈 배열일 때 처리가 없습니다.
→ for문은 cart.length가 0이면 자연스럽게 안 돌아가므로 별도 처리는 불필요합니다. total은 0이 됩니다.
