🛒 shop.js 테스트

shop.js 파일을 열어서 함수를 구현하고, 이 html 파일을 브라우저에서 열면 결과가 나옵니다.

개발자 도구 (F12 → Console)에서도 확인할 수 있어요.

💡 getStockLabel(stock) — "(품절)" 또는 "(재고: N개)"
💡 formatShopLine(index, name, price, stock) — "1. 떡 — 100원 (재고: 3개)"
💡 showShopList(items) — 전체 목록 문자열
💡 isAffordable(budget, price) — 예산 확인 → true/false
💡 buyItem(item, budget, cart) — 구매 처리 → 결과 메시지
💡 calcTotal(cart) — 장바구니 합산 → 총액
💡 showReceipt(cart) — 영수증 문자열 생성
💡 shopSession(items, budget) — 전체 흐름 (완성본)

🏷️ Function 1: getStockLabel

테스트 대기중...
테스트가 ❌이면 무슨 뜻인지 보기
"(품절)"이 안 나오면 — stock이 0일 때의 분기가 없습니다.
→ if (stock === 0) return "(품절)"; 을 먼저 처리하세요.

"(재고: 3개)"에서 숫자가 다르게 나오면 — stock 값을 문자열에 끼워넣지 않고 고정 숫자를 쓰고 있습니다.
→ "(재고: " + stock + "개)" 처럼 변수를 직접 넣어야 합니다.

괄호나 콜론, 공백이 틀리면 — 형식이 정확히 맞아야 합니다.
→ (재고: N개) — 여는 괄호, "재고", 콜론, 공백, 숫자, "개", 닫는 괄호 순서입니다.

📝 Function 2: formatShopLine

테스트 대기중...
테스트가 ❌이면 무슨 뜻인지 보기
"1. 떡 — 100원 (재고: 3개)"가 안 나오면 — 구분자나 공백이 다릅니다.
점(.), 줄표(—), "원" 뒤 공백을 정확히 확인하세요.
→ index + ". " + name + " — " + price + "원 " + getStockLabel(stock)

번호가 0부터 시작하면 — index를 그대로 쓰고 있습니다. showShopList에서 i+1을 넘겨줘야 합니다.
→ formatShopLine을 호출할 때 첫 번째 인자를 i + 1로 넘기세요.

getStockLabel 연동이 안 되면 — 재고 표시를 직접 만들고 있습니다.
→ 반드시 getStockLabel(stock)을 호출해서 그 결과를 이어붙이세요.

가격 뒤에 "원"이 없으면 — 숫자만 붙이고 "원"을 빠뜨렸습니다.
→ price + "원" 으로 붙여야 합니다.

📜 Function 3: showShopList

테스트 대기중...
테스트가 ❌이면 무슨 뜻인지 보기
상품이 하나만 나오거나 빠지면 — for문이 items 배열 전체를 돌지 않고 있습니다.
→ for (let i = 0; i < items.length; i++) 로 끝까지 돌아야 합니다.

번호가 0부터 시작하면 — i를 그대로 formatShopLine에 넘기고 있습니다.
→ formatShopLine(i + 1, ...) 으로 넘기세요.

줄바꿈(\n)이 없다고 나오면 — 항목들을 그냥 이어 붙이고 있습니다.
→ 각 항목 뒤에 "\n"을 붙여서 이어붙이세요.

품절 표시가 없으면 — formatShopLine이나 getStockLabel을 거치지 않고 직접 문자열을 만들고 있습니다.
→ formatShopLine에 stock 값을 제대로 넘겨주고 있는지 확인하세요. items[i].stock

원본 items 배열 부작용 검사가 실패하면 — items 배열을 수정하고 있습니다.
→ items는 읽기만 해야 합니다. push나 splice를 쓰지 마세요.

💰 Function 4: isAffordable

테스트 대기중...
테스트가 ❌이면 무슨 뜻인지 보기
딱 맞는 예산(300, 300)에서 false가 나오면 — >를 쓰고 있습니다. 등호가 빠졌습니다.
→ budget >= price 로 등호를 포함해야 합니다.

예산이 충분한데 false가 나오면 — 인자 순서가 바뀌었습니다. budget과 price를 거꾸로 비교하고 있습니다.
→ 함수 선언 isAffordable(budget, price) — budget이 첫 번째, price가 두 번째입니다.

반환값이 boolean이 아니면 — 숫자나 문자열을 return하고 있습니다.
→ return budget >= price; 비교 연산 자체가 true/false를 반환합니다.

🛍️ Function 5: buyItem

테스트 대기중...
테스트가 ❌이면 무슨 뜻인지 보기
구매 후 장바구니에 추가가 안 되면 — cart.push를 하지 않았거나 조건 분기 밖에서 push하고 있습니다.
→ 구매 성공 분기 안에서 cart.push(item) 을 호출하세요.

품절인데 "품절" 메시지가 안 나오면 — 재고 확인을 하지 않고 있습니다.
→ 가장 먼저 if (item.stock === 0) 으로 품절을 체크하고 메시지를 반환하세요.

품절인데 장바구니가 바뀌면 — 품절 체크 전에 cart.push를 하고 있습니다.
→ 품절 체크 → 예산 체크 → 구매 처리 순서로 작성하세요.

예산 부족인데 구매가 되면 — isAffordable을 호출하지 않거나 결과를 반대로 쓰고 있습니다.
→ if (!isAffordable(budget.gold, item.price)) 로 예산 부족일 때 early return하세요.

구매 후 재고가 안 줄어들면 — item.stock을 감소시키지 않았습니다.
→ 구매 성공 시 item.stock -= 1; 을 해야 합니다.

🧮 Function 6: calcTotal

테스트 대기중...
테스트가 ❌이면 무슨 뜻인지 보기
합계가 0으로 나오면 — total 누적을 하지 않고 있습니다.
→ let total = 0; 후 total += cart[i].price; 를 for문 안에서 반복하세요.

빈 장바구니에서 0이 안 나오면 — 초기값이 0이 아닌 다른 값으로 설정되어 있습니다.
→ let total = 0; 으로 시작해야 합니다.

합계가 더 크거나 작게 나오면 — cart[i].price 대신 다른 필드를 더하고 있습니다.
→ cart[i].price 를 더해야 합니다. cart[i].name이나 cart[i].stock이 아닙니다.

원본 cart 배열 부작용 검사가 실패하면 — cart 배열을 수정하고 있습니다.
→ cart는 읽기만 해야 합니다. splice나 pop을 쓰지 마세요.

🧾 Function 7: showReceipt

테스트 대기중...
테스트가 ❌이면 무슨 뜻인지 보기
문자열이 안 나오면 — return이 없거나 객체를 return하고 있습니다.
→ 영수증은 문자열 하나여야 합니다. return "..." 형태로 반환하세요.

상품명이 포함되지 않으면 — cart를 순회하지 않고 있습니다.
→ for문으로 cart를 돌면서 cart[i].name을 영수증 문자열에 이어붙이세요.

총액이 포함되지 않으면 — calcTotal을 호출하지 않았습니다.
→ let total = calcTotal(cart); 후 total을 영수증 문자열에 포함시키세요.

빈 장바구니 영수증에서 오류가 나면 — cart가 빈 배열일 때 처리가 없습니다.
→ for문은 cart.length가 0이면 자연스럽게 안 돌아가므로 별도 처리는 불필요합니다. total은 0이 됩니다.
